# flask项目

## 概要

- flask项目部署

## 建议

- flask可以是一个优秀的后台服务器，只用来处理后台接口请求
- 最基础的配置就是动静分离，静态文件通过nginx来处理，接口下发到flask来处理和返回
- 这种最好的处理方式就是单页面应用，最开始就下载好基础前端框架和页面还有js/css文件
- 之后都是通过接口请求来获取页面代码块，这个要如何更新
