# restful_api

## 概要

- 网络应用分为前端和后端,前端设备现在层出不穷,因此必须要有一种统一的机制,方便不同的前端设备和后端进行通信.这导致API架构的流行.
- RESTFul API是目前比较成熟的互联网应用程序的API设计理论.

## 设计好的RESTful api

- 协议: 使用HTTPs协议
- 域名: 应该尽量将API部署在专用域名下"https://api.example.com"
- 版本: 应该将版本放入URI"https://api.example.com/v1/"
